 Phase 4: Key Architectural Decisions

 Theming (CSS Custom Properties + TailwindCSS v4)

 1. loadTheme.ts reads theme.json server-side
 2. Values injected as CSS custom properties in layout.tsx via <style> tag
 3. TailwindCSS v4 @theme block maps vars to utility names (text-accent, bg-surface)
 4. Dark/light mode via next-themes with attribute="class" — swaps CSS var values in .dark {}

 Data Loading

 - DATA_SOURCE env var controls source: empty = local data/data.json, or https:// URL
 - loadData.ts marked import "server-only" — called in async page.tsx Server Component
 - Data passed to DataProvider context; all components read via useData() hook

 Component Boundary Strategy

 - page.tsx = async Server Component (fetches data, no useEffect)
 - All section components = Client Components ("use client") for animations/interactivity
 - Keep "use client" as deep as possible to maximize server rendering

 Scroll Spy

 - useScrollSpy.ts uses IntersectionObserver (threshold 0.6) watching section id attributes
 - Active section stored in context, read by NavLink for accent + underline highlight

 Skill Bar Animation

 - useAnimateOnView.ts wraps IntersectionObserver → adds animate-in class on entry
 - Bar fill is a CSS transition on --bar-width custom property (no JS animation)
